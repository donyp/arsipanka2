🚀 QUICK 7-STEP DEPLOYMENT GUIDE
════════════════════════════════════════════════════════════════════════════════

1. CREATE SPACE
   → Go to https://huggingface.co/spaces/create
   → Choose Docker SDK
   → Name: pusat-arsip-anka

2. SETUP GIT
   → git config user.email "your@email.com"
   → git config user.name "Your Name"
   → git remote add hf https://huggingface.co/spaces/USERNAME/pusat-arsip-anka

3. DEPLOY
   → git add .
   → git commit -m "Deploy: Pusat Arsip Anka"
   → git push -u hf main

4. GENERATE JWT_SECRET
   → Run: node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
   → Save the output

5. SET ENVIRONMENT VARIABLES
   → Go to Space Settings > Variables and Secrets
   → Add SUPABASE_URL (from Supabase dashboard)
   → Add SUPABASE_SERVICE_ROLE_KEY (from Supabase)
   → Add JWT_SECRET (from step 4)
   → Add PORT=7860 and NODE_ENV=production

6. WAIT FOR BUILD
   → Monitor Space > App logs
   → Should take 2-5 minutes
   → Wait for "Running" status

7. ACCESS APP
   → https://YOUR_USERNAME-pusat-arsip-anka.hf.space

   • Auto-restarts every 48 hours without activity
   • Manual restart available in HF Settings

   • First request might be slower (container initialization)
   • Subsequent requests normal speed
   • This is expected HF Spaces behavior

🐛 COMMON ISSUES & QUICK FIXES
════════════════════════════════════════════════════════════════════════════════

Build Fails:
  FIX: Check .dockerignore, reduce total size, verify Dockerfile syntax

App Won't Start:
  FIX: Verify PORT=7860, check start.sh permissions, review logs

Database Connection Error:
  FIX: Verify SUPABASE_URL and key, check Supabase project active

500 Errors:
  FIX: Check JWT_SECRET set, verify credentials, check console logs

Memory Issues:
  FIX: Disable Alist if not needed, clear temp files, optimize queries
